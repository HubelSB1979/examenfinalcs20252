# Examen Parcial – Construcción de Software  
## Sistema de Ventas Web (HTML / CSS / JavaScript + Git)

### Curso
Construcción de Software

### Modalidad
Trabajo grupal – Evaluación práctica

---

## 1. Contexto del examen

Usted forma parte de un equipo de desarrollo de software contratado para **mejorar un sistema de ventas web** que actualmente se encuentra en una versión inicial (prototipo).

El cliente ha entregado una **base funcional mínima**, pero solicita **correcciones, mejoras y nuevas funcionalidades**, las cuales deberán ser implementadas respetando buenas prácticas de ingeniería de software y trabajo colaborativo usando **Git**.

---

## 2. Objetivo del examen

- Analizar un código existente.
- Corregir errores y deficiencias.
- Implementar nuevas funcionalidades solicitadas por el cliente.
- Trabajar colaborativamente usando control de versiones (Git).
- Evidenciar buenas prácticas de desarrollo de software.

---

## 3. Alcance técnico

El sistema está desarrollado únicamente con:

- HTML
- CSS
- JavaScript (Vanilla JS)
- Almacenamiento local (`localStorage`)
- Git (repositorio remoto)

Si el grupo lo considera pertinente, puede usar cualquier otra tecnologia como ReactJS, VueJS, AngularJS, Laravel, etc, siempre y cuando se cumplan los requisitos

Estructura basica del proyecto

├── index.html # Login
├── inicio.html # Panel principal
├── vender.html # Registro de ventas
├── css/
│ └── styles.css
├── js/
│ ├── app.js # Login
│ ├── layout.js # Sesión, menú, logout
│ └── vender.js # Lógica de ventas
